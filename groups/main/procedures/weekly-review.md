# Weekly Review

Friday project review. Your principal scans this over the weekend to plan the week ahead.

Use `mcp__time__*` tools for ALL date/time computation. Never calculate dates yourself.

## Execution

1. Read MEMORY.md for the file index — every project file listed there is in scope
2. For each project file, check email and calendar for recent activity
3. Update the file with current status, blockers, and last-checked date
4. Update MEMORY.md if any files were created or removed
5. Compose one message with what your principal needs to see. Send via `mcp__nanoclaw__send_message`, even if everything is green

## What to Surface

In the message to your principal:
- Stalled or blocked projects get a one-line status each — what's wrong and what you recommend
- Healthy projects get skipped. If everything is healthy, say so
- If a project has been stalled for multiple weeks, escalate the framing — it's drifting, not just "needs attention"

## Decisions Needed

If any project is blocked on your principal's input, surface it here. Format the same way as the morning briefing — numbered list, recommendation bold, deadline if any.

If nothing needs a decision, skip this section entirely.

## Tone

Write like a weekly staff memo, not a dashboard dump. Judgment-forward — don't just list what's overdue, say what matters and what you recommend. Terse, direct, scannable in 2 minutes.

## Output

Your final response text must be completely empty — output nothing. All communication happens exclusively through MCP tools.
